2. Analysis of Differences

A. BMD Modeling Tool

Root cause of numerical differences. The reference used BMDS 2.7.0 (via pybmds 0.11) with 15+ model types (Linear, Polynomial 2°–8°, Power, Hill, Exponential M2–M5). We use BMDExpress 3 which fits fewer models (Hill, Power, Linear, Polynomial 2°, Exponential M2–M5). Fewer candidate models means fewer chances to find a well-fitting curve, and different model selection criteria (AIC vs best-fit logic) yield different BMD/BMDL values.

C. Status Classification (NVM vs UREP)

Systematic mismatch. The reference has 4 UREP endpoints and 1 NVM. We have 0 UREP and 8 NVM. This happens because BMDExpress 3 model fitting either succeeds (viable) or fails entirely (NVM); it rarely produces the intermediate state where a model fits but BMDU/BMDL > 40 (UREP). BMDS 2.7.0 with its richer model set more often finds some model that fits but with unreliable confidence intervals, yielding UREP.

D. LOEL / NOEL Discrepancies

Male T4: Reference LOEL = 12, NOEL = 4. Ours: LOEL = 37, NOEL = 12. The reference likely used different p-value thresholds or applied outlier exclusion (Dixon and Massey test, per their methods section) which we do not currently perform. This shifts which dose first reaches significance.
Female Manual Hematocrit: Reference LOEL = 37, NOEL = 12. Ours: LOEL = — (none), NOEL = 111. The reference found pairwise significance at 37 mg/kg; our Dunnett/Williams tests did not flag any dose as pairwise-significant for this endpoint.

E. Direction of Change

Bug: direction field is empty for all rows. The Jonckheere test computes direction (1 = UP, −1 = DOWN), but it is not propagating into the BMD summary. The reference has clear DOWN/UP annotations for 4 of 10 endpoints. This is a code bug in build_table_data().

The reference included only endpoints with significant trend AND pairwise tests. Cholesterol and Sorbitol Dehydrogenase were included despite UREP because their LOEL/NOEL are informative.

4. Methodology Notes

NIEHS Reference (BMDS 2.7.0)

  • Tool: pybmds 0.11 (Python BMDS interface)
  • Models: Linear, Poly 2°–8°, Power, Hill, Exp M2–M5
  • BMR: 1 SD relative to control
  • Model selection: EPA guidance — lowest AIC or lowest BMDL
  • Bins: Failure → NVM → NR → UREP → Viable
  • NR threshold: BMD < ⅓ lowest nonzero dose
  • UREP: subject matter expert review
  • Stats: Williams/Dunnett (parametric), Shirley/Dunn (nonparametric)
  • Outlier exclusion: Dixon & Massey test

5dToxReport (BMDExpress 3)

  • Tool: BMDExpress 3 Java API (native BMD pipeline)
  • Models: Hill, Power, Linear, Poly 2°, Exp M2–M5
  • BMR: 1 SD relative to control
  • Model selection: bestBMD from probeStatResults
  • Bins: Failure → NVM → NR → UREP → Viable
  • NR threshold: BMD < ⅓ lowest nonzero dose
  • UREP: BMDU/BMDL > 40 (automated, no SME review)
  • Stats: Jonckheere (Python) + Williams/Dunnett (Java, sciome-commons-math)
  • Outlier exclusion: none currently
