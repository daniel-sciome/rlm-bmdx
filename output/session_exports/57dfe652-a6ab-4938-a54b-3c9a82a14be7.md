# Plan: Gene Function Crawl + Pathway Enrichment + Hybrid Scoring

## Context

The current pipeline finds genes mentioned in toxicogenomics papers but misses gene function papers — papers that explain WHY a gene matters for organ-specific toxicity but don't use tox vocabulary (e.g., "MFN2 regulates mitochondrial fusion in cardiomyocytes"). Three complementary approaches address this gap.

## Feature 1: Gene-Function Crawl (`genefunc_crawl.py` — new file)

A second-pass S2 crawl that searches for each consensus/moderate gene's biological function in its associated organs.

**Approach:**
- Load 69 consensus+moderate genes from `citegraph_output/gene_consensus.json`
- For each gene-organ pair, search S2 (e.g., "MFN2 mitochondrial cardiomyocyte function")
- Different relevance scoring: gene name in title/abstract + organ keyword = pass (no tox vocabulary required)
- 1-hop expansion from top search results to catch related function papers
- Budget: 400 papers, 200 API calls
- Output: `citegraph_output_genefunc/papers.json`, `edges.json`, `citation_graph.gml`

**Key functions:**
- `load_gene_organ_pairs(consensus_path) -> list[tuple[str, list[str]]]` — loads genes + their organs
- `build_search_queries(gene, organs) -> list[str]` — generates search strings like "NFE2L2 oxidative stress liver mechanism"
- `gene_relevance(paper, target_gene, target_organs) -> float` — scores based on gene name + organ mention, not tox keywords
- `run_genefunc_crawl(consensus_path, max_papers=400)` — orchestrator

**After crawl:** Run `python extract.py citegraph_output_genefunc` to extract genes/claims via LLM (must wait for current organ extractions to finish).

## Feature 2: Pathway Enrichment (`pathway_enrich.py` — new file)

Cross-reference our genes with KEGG and Reactome pathway databases.

**KEGG (REST API, confirmed working):**
- `rest.kegg.jp/link/hsa/pathway` — all human gene-pathway links (bulk download)
- `rest.kegg.jp/link/rno/pathway` — all rat gene-pathway links
- `rest.kegg.jp/list/pathway/hsa` — pathway ID → name mapping
- `rest.kegg.jp/conv/hsa/uniprot` + `rest.kegg.jp/list/hsa` — KEGG gene ID → gene symbol mapping
- Format: tab-separated, parse with csv module

**Reactome (REST API, confirmed working):**
- `reactome.org/ContentService/search/query?query=GENE_NAME&species=Homo+sapiens&types=Pathway` — per-gene pathway lookup
- Rate limit: ~70 calls for consensus+moderate genes, ~1s each

**Output:** `pathway_enrichment.tsv` with columns: `gene_symbol, kegg_pathway_id, kegg_pathway_name, reactome_pathway_id, reactome_pathway_name, tox_evidence, tox_organs, species`

**Key functions:**
- `fetch_kegg_pathways(species="hsa") -> dict[str, list[tuple[str, str]]]` — gene_symbol → [(pathway_id, pathway_name)]
- `fetch_reactome_pathways(gene_symbol) -> list[tuple[str, str]]` — [(pathway_id, pathway_name)]
- `build_pathway_tsv(consensus_path, output_path)` — orchestrator
