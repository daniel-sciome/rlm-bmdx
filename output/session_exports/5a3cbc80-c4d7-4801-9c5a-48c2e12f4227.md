# bmdx Dose-Response Interpretation Engine

## Context

bmdx has a rich toxicogenomics knowledge base (2,315 papers, 161 consensus genes, 130K GO mappings, 2,860 pathway entries, 4,431 extracted claims) but no way to **interpret user data** against it. The user wants to take a dose-response dataset (gene-level BMD results from BMDExpress) and get a grounded, literature-backed narrative answering: "What do these dose-response patterns tell us about organism/organ-level effects?"

This is the same architectural pattern as ai-litmine's `interpret.py` — RAG over a literature-derived KB — but adapted for toxicogenomics dose-response data.

## interpret.py Architecture

### 1. `ToxKBQuerier` — DB query helpers

Wraps bmdx.duckdb (read-only connection). Methods:

- `gene_pathways(gene) → [{pathway_db, pathway_id, pathway_name, species}]`
  - Query: `SELECT * FROM pathways WHERE gene_symbol = ?`
- `gene_go_terms(gene) → [{go_id, go_term, cluster_id}]`
  - Query: `SELECT g.go_id, t.go_term, t.cluster_id FROM gene_go_terms g JOIN go_terms t ON g.go_id = t.go_id WHERE g.gene_symbol = ?`
- `gene_organs(gene) → [str]`
  - Query: `SELECT organs FROM genes WHERE gene_symbol = ?`
- `gene_papers(gene) → [{paper_id, title, year, citation_count}]`
  - Query: `SELECT p.* FROM papers p JOIN paper_genes pg ON p.paper_id = pg.paper_id WHERE pg.gene_symbol = ?`
- `gene_claims(gene) → [{claim, paper_title, year}]`
  - Query: `SELECT pc.claim, p.title, p.year FROM paper_claims pc JOIN paper_genes pg ON pc.paper_id = pg.paper_id JOIN papers p ON pc.paper_id = p.paper_id WHERE pg.gene_symbol = ?`
- `pathway_genes(pathway_name) → [str]`
  - Query: `SELECT DISTINCT gene_symbol FROM pathways WHERE pathway_name = ?`
- `all_pathway_gene_counts() → {pathway_name: gene_count}`
  - For Fisher's exact test background
- `total_gene_count() → int`
  - Query: `SELECT COUNT(DISTINCT gene_symbol) FROM genes`
- `gene_evidence(gene) → {evidence, mention_count}`
  - Query: `SELECT evidence, mention_count FROM genes WHERE gene_symbol = ?`

### 2. `enrich_pathways()` — Pathway enrichment with Fisher's exact test

For each pathway represented in the user's responsive gene list:

```
                In pathway    Not in pathway
Responsive       a              b
Not responsive   c              d
```

- `a` = user genes in this pathway
- `b` = user genes NOT in this pathway
- `c` = background genes in this pathway minus a
- `d` = background genes NOT in this pathway minus b
- Background = all genes in `pathways` table
- Use `scipy.stats.fisher_exact(alternative='greater')` for over-representation
- Return sorted by p-value, with BH FDR correction

### 3. `enrich_go_terms()` — GO term enrichment (same pattern)

Same Fisher's exact test but against the `gene_go_terms` table. Group by GO term.

### 4. `compute_bmd_ordering()` — Dose-ordered pathway narrative

- Group responsive genes by enriched pathway (significant pathways only)
- For each pathway: median BMD, mean BMD, min BMD, direction summary
- Sort pathways ascending by median BMD
- Output: ordered list of `{pathway, median_bmd, gene_count, direction, genes}`

### 5. `compute_organ_signature()` — Tissue-specificity enrichment

- For each responsive gene, get organ annotations from `genes.organs`
- Count organ occurrences across all responsive genes
- Normalize by total organ frequency in the KB (to avoid liver bias from crawl composition)
- Rank organs by enrichment ratio
- Output: `{organ: score}` dict

### 6. `build_interpretation_context()` — Assemble structured context

Combines all of the above into a text block for the LLM:

```
=== DOSE-RESPONSE SUMMARY ===
N responsive genes, BMD range X-Y
Direction: N up, M down

=== PATHWAY ENRICHMENT (FDR < 0.05) ===
1. Nrf2 oxidative stress (p=1.2e-8, 15/30 genes)
   Genes: NFE2L2, KEAP1, HMOX1, GCLC, ...
2. p53 apoptosis (p=2.1e-6, 8/25 genes)
   ...

=== DOSE-ORDERED RESPONSE ===
BMD 0.1-1.0: Nrf2 pathway (median BMD 0.4, 15 genes, mostly UP)
BMD 1.0-5.0: UPR/ER stress (median BMD 2.1, 6 genes, mostly UP)
BMD 5.0-20:  Apoptosis (median BMD 8.3, 8 genes, mixed)

=== ORGAN SIGNATURE ===
Liver: 2.4x enriched (CYP1A1, PPARA, AHR, GCLC)
Kidney: 1.1x enriched (HAVCR1, CLU)

=== LITERATURE CONTEXT ===
## NFE2L2 (81 papers, consensus gene)
  Organs: liver, kidney, lung, brain, heart
  Key claims:
  - "NRF2 activation reduces oxidative stress..." (Sutherland 2017)
  - ...
## TP53 (63 papers, consensus gene)
  ...

=== TOP RELEVANT PAPERS ===
[Papers discussing multiple responsive genes, sorted by citation count]
```

### 7. LLM Synthesis

System prompt: toxicogenomics expert analyst
Interpretation prompt asks the LLM to generate a report covering:
1. **Biological response narrative** — what's happening at each dose level
2. **Organ-level prediction** — which organs are most likely affected and why
3. **Mechanism of action** — what pathways/processes suggest about the toxicant's mode of action
4. **Protective vs. adverse responses** — which responses are adaptive (NRF2) vs. damage (apoptosis)
5. **Literature support** — citations grounding each conclusion
6. **Confidence assessment** — how many genes have strong literature support vs. novel findings

### 8. CLI

```
python interpret.py <dose_response.csv> [--db bmdx.duckdb] [--output output/interpretation.md] [--fdr 0.05]
```
