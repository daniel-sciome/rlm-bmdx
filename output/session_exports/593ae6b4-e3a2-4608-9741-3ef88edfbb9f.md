### Core Architecture

The RLM pattern fundamentally reframes how LLMs handle long contexts. Instead of forcing lengthy inputs into fixed context windows, **the model treats the prompt as an external environment** that it can programmatically examine and decompose.

**Key mechanism:**
- The LLM operates as a **programmer** within a Python REPL-like notebook environment
- The entire context is pre-loaded as a variable in memory (not fed token-by-token)
- The model writes code to inspect, search, and chunk text based on structure

1. **Decomposition**: Instead of reading the whole prompt linearly, the model writes code (regex patterns, chunking logic, searches) to break the context into meaningful pieces
2. **Recursion**: The model calls itself on individual snippets, accumulating understanding piece by piece
