# Phase 2 Crawl: Interpretation-Driven Literature Expansion

## Motivation

The bmdx pipeline has completed a full cycle: 2,319 papers crawled, 161 consensus
genes identified, a DuckDB knowledge base built, and dose-response interpretation
narratives generated by 5 independent AI models (qwen2.5:14b, gemma2:9b,
claude-haiku-4-5, claude-sonnet-4-6, claude-opus-4-6) across 3 runs each.

The 15 resulting narratives are remarkably consistent on core findings — NRF2-mediated
oxidative stress response at low doses, TP53/BAX-driven apoptosis at mid-range,
NF-kB/TNF inflammation at high doses — but they also reveal systematic gaps in the
knowledge base. Every model hedges on the same questions. Every model flags the same
mechanisms as poorly supported. The disagreements between models cluster around
exactly those areas where the KB has thin coverage.

This is the key insight: **the interpretation pipeline is a gap detector.** Where
models agree, the KB is working. Where they disagree or express low confidence, the
KB needs more literature. We can use the interpretation output to design targeted
crawls that fill precisely the holes the models found, then re-interpret and repeat.

The pipeline becomes a feedback loop:

    crawl → extract → build KB → interpret → identify gaps → crawl again

## What the Narratives Reveal

### The Central Unanswered Question

All 5 models identify an adaptive-to-adverse transition — the dose at which
protective NRF2/antioxidant responses are overwhelmed and cells commit to apoptosis,
inflammation, and tissue damage. This is the single most important question for
regulatory toxicology. But the models cannot agree on where it falls:

| Model              | Transition BMD  | Confidence |
|--------------------|-----------------|------------|
| claude-sonnet-4-6  | 2.5–5.5         | Moderate   |
| claude-opus-4-6    | 4.0–6.0         | Moderate   |
| claude-haiku-4-5   | 0.3–5.5 (broad) | Low        |
| qwen2.5:14b        | "moderate dose"  | Low — explicitly flagged |
| gemma2:9b          | "moderate doses" | Low — "needs more data" |

The local models cannot even give specific BMD values. The KB lacks papers that
empirically characterize dose thresholds where antioxidant defenses become
overwhelmed.

### Mechanisms Identified but Unsupported

The models describe mechanisms they recognize from training data but which have

### Model Disagreements as Gap Indicators

Where models disagree, the KB evidence is insufficient to constrain interpretation:

1. **Inflammatory onset dose**: Haiku says BMD 0.3, Sonnet says 2.5, Opus says 4.0
2. **Brain as target organ**: Only Opus provides sub-regional predictions; others ignore it
3. **Heart as target organ**: Gemma calls it "novel" and "needs validation"
4. **BCL2 upregulation meaning**: Haiku says "failed compensatory"; Sonnet says "apoptotic priming"; Opus says "cellular conflict"
5. **Intestine relevance**: Only the two largest Claude models mention it at all


## Crawl Design

### Strategy: Three Crawl Tiers

Rather than one monolithic crawl, we run three tiers targeting different gap types.
Each tier uses a different crawl strategy matched to the kind of gap it fills.

### Tier 1 — Mechanism-Specific Crawls

**Goal:** Fill the zero-paper mechanism gaps with dedicated per-mechanism crawls.
These are the mechanisms every model identifies but the KB cannot support.

Each mechanism gets its own crawl config with targeted seed papers and search queries.
These use the existing `citegraph.py` architecture with custom configs, similar to the
organ-specific crawls.

**Crawl configs:**

#### 1a. NRF2 Dose-Response Dynamics (max_papers: 400)

The core gap. Need papers that empirically measure how NRF2 signaling changes
across a dose gradient, and where the protective-to-maladaptive switch occurs.

Seeds (search-derived):
- "NRF2 dose response oxidative stress threshold"
- "KEAP1 NRF2 activation dose dependent"
- "NRF2 antioxidant response saturation toxicity"
- "oxidative stress adaptive adverse dose transition"
- "NRF2 biphasic hormesis oxidative stress"

Boost keywords:
```
nrf2, keap1, nfe2l2, oxidative stress, dose-response, dose-dependent,
threshold, biphasic, hormesis, adaptive, adverse, transition, saturation,
antioxidant response element, ARE, electrophile, sulforaphane
```

#### 1b. Inflammasome and Inflammatory Thresholds (max_papers: 300)

NLRP3 is in the dose-response data (BMD 13.0) and every model discusses
inflammation, but the KB has no papers on inflammatory dose-response kinetics.

Seeds (search-derived):
- "NLRP3 inflammasome activation dose response"
- "NF-kB activation threshold toxicology dose"
- "inflammation onset dose chemical toxicity"
- "TNF IL6 dose dependent toxicogenomics"

Boost keywords:
```
nlrp3, inflammasome, nfkb, nf-kb, il-1beta, il1b, caspase-1,
pyroptosis, dose-response, threshold, inflammation, inflammatory,
innate immune, damage-associated, DAMP, sterile inflammation
```

#### 1c. Cell Death Pathway Decision Points (max_papers: 400)

The narrative models recognize ferroptosis, necroptosis, and autophagy as
distinct from classical apoptosis, but the KB treats all cell death as
BAX/BCL2/CASP3 apoptosis. Need papers that distinguish cell death modes
in a dose-dependent context.

Seeds (search-derived):
- "ferroptosis dose response oxidative stress"
- "ferroptosis apoptosis necroptosis decision"
- "autophagy apoptosis switch dose dependent"
- "necroptosis toxicology dose response"
- "cell death mode determination dose"
- "GPX4 ferroptosis threshold"

Boost keywords:
```

gpx4, ripk1, ripk3, mlkl, beclin, sqstm1, p62,
cell death, programmed, regulated, dose-dependent, threshold,
lipid peroxidation, iron, erastin
```

**Goal:** For specific genes flagged as lacking evidence, run targeted searches

| SQSTM1 | 3.0  | p62 SQSTM1 NRF2 autophagy oxidative stress        | 100        |
| STAT3  | 8.5  | STAT3 JAK-STAT toxicology inflammation dose        | 100        |
| SIRT1  | 2.5  | SIRT1 NAD oxidative stress dose response toxicity  | 100        |

### Implementation

The crawl configs follow the existing `ORGAN_CRAWL_CONFIGS` pattern in
`citegraph.py`. Tier 1 and Tier 2 configs are added to a new
`MECHANISM_CRAWL_CONFIGS` and `ORGAN_CRAWL_CONFIGS` (extended) dictionary.
Tier 3 extends `genefunc_crawl.py` with a tox-focused query template set.

All crawls write to separate output directories (`citegraph_output_nrf2dose/`,
`citegraph_output_inflammasome/`, etc.) and are merged into the unified
`bmdx.duckdb` via the existing `build_db.py` pipeline.

The crawls can run sequentially overnight, or in parallel across the two
machines (one crawling, one extracting completed outputs).

### After the Crawl

1. Extract all new papers via `extract.py` across both GPUs
2. Rebuild `bmdx.duckdb` via `build_db.py` with all crawl directories
3. Re-run the interpretation pipeline with all 5 models
4. Run concordance analysis on the new narratives
5. Compare: did the gaps close? Did the models converge on the transition dose?
   Did new gaps emerge?
6. If significant gaps remain, repeat the cycle

This is the feedback loop. The interpretation pipeline is not just an output —
it is an instrument for measuring the quality of the knowledge base.
