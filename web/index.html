README FIRST, OR SUFFER THE CONSEQUENCES

Work through the process slowly and deliberately, following instructions precisely. It should work well. All bets are off if you move too quickly, or jump around. User safeguards have not been explored yet. This trial run is not to give it a stress test, but to successfully generate results.

5dToxReport builds structured toxicology reports for NTP “5 Day Genomic Dose Response in Sprague-Dawley Rats” studies. You work through sections in order: identify the chemical, generate text with an LLM, upload dose-response data, review and approve each section, then export the final report as PDF.

Every section follows the same cycle: generate → review → edit (optional) → approve. Approved sections are saved to the server and restored automatically if you reload the page or come back later. You can unapprove any section to regenerate or re-edit it.

Step 1 — Identify the Chemical

Type any identifier into the Chemical Identification form: a chemical name, CASRN, DTXSID, or PubChem CID. The app resolves it against EPA’s CompTox and PubChem databases and fills in all the other fields automatically.

  1. Enter a value in any one field (e.g., 95-50-1 in the CASRN box).
  2. The app auto-resolves on blur — all fields populate within a few seconds.
  3. If a saved session exists for this DTXSID, it loads automatically: all previously approved sections reappear exactly as you left them.
The DTXSID is the session key. Everything you approve is saved under this identifier. If you switch chemicals, you start (or resume) a different session.

Step 2 — Generate the Background

Once the chemical is resolved, click Generate Background. The server gathers data from multiple sources (ATSDR ToxProfiles, PubChem, EPA CompTox, the bmdx knowledge base) and sends it to Claude, which writes a structured six-paragraph background section with inline references.

  1. Select an LLM from the dropdown (Claude is recommended; Ollama is for local/offline use).
  2. Click Generate Background. A progress panel shows real-time status as data sources are queried.
  3. The generated text appears as editable paragraphs. References are listed below.
  4. Edit the text directly in the browser if you want to change wording, fix facts, or rearrange paragraphs.
  5. Click Approve to save. If you edited the text before approving, the app learns from your changes and adjusts its writing style for future generations.

After approval, the Background tab gets a green border and a version badge. You can click Edit to unlock it again, or Try Again to regenerate from scratch.

Step 3 — Upload Dose-Response Data

The Data tab appears after the background section exists. This is where you upload dose-response data files. All files go into a shared pool.

  1. Drag files onto the drop zone, or click to browse. You can upload individual files or a .zip archive containing multiple files.
  2. Upload any and all .bm2, .csv, .txt, and .xlsx files that contain dose-response data on your test article of interest that you deem relevant. The app will fingerprint each file, classify its data type, and merge everything into a unified pool.
  3. Uploaded files appear in the collapsible Uploaded files list with type and row count.

Step 4 — Validate & Integrate

Once files are uploaded, the Pool Validation panel appears. This step merges your files into a single integrated dataset, resolving overlaps between files that cover the same experimental domain. The entire dataset-building pipeline — file parsing, domain classification, dose-response extraction, and BMD model results — runs through BMDExpress 3 native code (Java subprocess), ensuring full fidelity with the BMDExpress analysis engine.

  1. Click Preview to inspect the raw data extracted from each file before committing to integration.
  2. Click Validate & Integrate. The server fingerprints each file, detects its domain (body weight, organ weights, clinical chemistry, etc.), checks for conflicts, and merges everything.
  3. A coverage matrix appears showing which domains are covered and by which file tier (.bm2 takes priority over .txt/.xlsx).
  4. If there are conflicts (two files claiming the same domain), resolve them using the controls shown.
  5. Review the validation results, then click Approve to lock the pool.

After pool approval, the app automatically processes all files: apical endpoint tables are generated from the integrated data, and genomics analysis runs on gene expression data. This can take 10–30 seconds depending on file size.

Step 5 — Review Apical Endpoint Results

The Apical tab shows one card per experimental domain (e.g., “Organ and Body Weights”, “Clinical Pathology”). Each card contains:

  • NTP-style dose-response tables with statistical significance markers (* p<0.05, ** p<0.01)
  • An LLM-generated narrative summarizing the key findings (direction, magnitude, dose-response relationship)
  • BMD/BMDL values for each endpoint where modeling succeeded
  1. Review each card. Edit the narrative text if needed.
  2. Click Approve on each card individually.
  3. As you approve cards, the BMD Summary tab auto-populates with a cross-domain summary table. Approve that too.

Step 6 — Configure GO Category Filters

Before generating the genomics report, review the GO category filter settings. Click the gear icon (top right) to open the settings panel. Here you can adjust:

  • Minimum and maximum gene counts per GO category
  • Minimum BMD percentage cutoff
  • Which BMD statistics to display in the tables

Click Apply & Reprocess to save your settings. You can change these later and reprocess at any time.

Step 7 — Review Genomics Analysis

The Genomics tab shows cards organized by organ and sex (e.g., “Liver (Male)”). Each card contains:

  • A GO Biological Process table ranking gene categories by their median (or other selected statistic) BMD value. Categories with the lowest BMD are the most sensitive biological pathways.
  • A Top Genes table showing individual genes ranked by BMD.
  • An LLM-generated narrative interpreting the pathway results in the context of the test article’s known toxicology.
  1. Click Generate Report to run the genomics analysis for the first time. This produces the GO and gene tables and generates the LLM narrative for each card.
  2. Review the tables and narrative for each organ/sex combination.
  3. Edit the narrative if needed.
  4. Approve each card.

Step 8 — Generate Materials & Methods

The Methods tab generates the Materials and Methods section. It pulls study parameters from the uploaded data (species, strain, doses, route, duration, endpoints measured) and writes structured prose covering the experimental design, BMD modeling approach, and statistical methods.

  1. Click Generate. The server assembles context from the approved sections and uploaded data, then calls the LLM.
  2. Review and edit the generated text.
  3. Approve.

You can click Preview Data to see exactly what context the LLM will receive before generating.

Step 9 — Generate Summary

The Summary tab synthesizes all approved sections into a concise summary. It reads the approved background, apical results, genomics findings, and BMD summary to produce a cohesive conclusion.

  1. Click Generate.
  2. Review, edit if needed, approve.

Step 10 — Export

The Report tab contains an integrated PDF viewer that shows a live preview of your report, updated automatically as you approve sections. When you are satisfied with the report, click Export PDF to download a tagged PDF/UA-1 generated with Typst.

How Sessions Work

Every time you approve a section, it’s saved to the server as a JSON file under your chemical’s DTXSID. When you return and resolve the same DTXSID, all approved sections are restored automatically.

  • Version history — each approve archives the previous version. Click the version badge (e.g., “v3 ▾”) to browse and restore past versions.
  • Unapprove — click Try Again on any approved section to delete the saved data and regenerate.
  • Style learning — when you edit text before approving, the app extracts style rules from your changes. These persist in a global style profile and are applied to future LLM generations.
Sessions live on the server’s filesystem. On Cloud Run, containers are ephemeral — if the container restarts, sessions created since the last deployment are lost. This is expected. Sessions baked into the image at deploy time always survive.

Settings

Click the gear icon in the top-right corner to open the settings panel. Settings are saved in your browser and persist across page reloads.

SettingWhat it controls
BMD StatisticsWhich aggregate statistics (mean, median, 5th percentile, etc.) appear in GO category tables. Each checked statistic produces a separate table.
Dose UnitUnit shown in table headers and narratives (e.g., mg/kg). Auto-detected from uploaded files when possible.
Significance ThresholdsP-value cutoffs for * and ** markers in report tables.
GO Category FiltersMin/max gene counts and BMD percentage cutoffs that determine which GO categories appear in genomics tables.

After changing settings, click Apply & Reprocess to regenerate tables with the new parameters.
